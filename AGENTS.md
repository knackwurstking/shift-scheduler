# Shift Scheduler

## Design Philosophy
- **Go WASM first** ‚Äî Shift as much logic as possible into Go WASM to minimize JavaScript. JS is only used for bootstrapping (loading WASM, DOM querying). All interactivity (swipe, date picker, theme toggle, grid rendering) should eventually move into Go WASM.
- **Server-side rendering** via `html/template` for initial page load, then Go WASM handles all client-side behavior.
- **Migrating from a-h/templ** ‚Äî old `.templ` files remain in `templates/.pages/` and `templates/.components/` as backup.

## Stack
- **Go 1.25** with Echo web framework
- **html/template** (migrating from a-h/templ)
- **Go WASM** frontend ‚Äî all client-side interactivity driven from Go

## Build Commands
```bash
make init    # Run templ generate (legacy), copy wasm_exec.js, go mod tidy
make build   # Compile WASM + server binary
make run     # init + build + start server on :3000
```
Note: `make generate` still runs `templ generate` on old `.templ` files ‚Äî this will be removed once migration is complete.

## Architecture
- `cmd/shift-scheduler/` ‚Äî Echo web server (serves `public/` as static)
- `cmd/wasm/` ‚Äî WASM binary (all frontend logic: swipe, date picking, theme, calendar grid)
- `templates/` ‚Äî `index.html` (new, `html/template`) + `.pages/` and `.components/` (old templ backup)
- `internal/handlers/` ‚Äî HTTP handlers
- `internal/localization/` ‚Äî i18n (en/de via Accept-Language header)
- `public/` ‚Äî Static assets served at `/` (WASM, CSS, icons)
