## Design Philosophy
- **Go WASM first** — Shift as much logic as possible into Go WASM to minimize JavaScript. JS is only used for bootstrapping (loading WASM, DOM querying). All interactivity (swipe, date picker, theme toggle, grid rendering) should eventually move into Go WASM.
- **Server-side rendering** via `html/template` for initial page load, then Go WASM handles all client-side behavior.
- **Migrating from a-h/templ** — old `.templ` files remain in `templates/.pages/` and `templates/.components/` as backup.

## Stack
- **Go 1.25** with Echo web framework
- **html/template** (migrating from a-h/templ)
- **Go WASM** frontend — all client-side interactivity driven from Go

## Build Commands
```bash
make init    # Copy wasm_exec.js from GOROOT, go mod tidy
make build   # Compile WASM + server binary
make run     # build + start server on :3000
make dev     # build + start server with VERBOSE=true on :3000
```

## Architecture
- `cmd/shift-scheduler/` — Echo web server (serves `public/` as static)
- `cmd/wasm/` — WASM binary (all frontend logic: swipe, date picking, theme, calendar grid)
- `templates/` — `index.html` (new, `html/template`) + `.pages/` and `.components/` (old templ backup)
- `internal/handlers/` — HTTP handlers
- `internal/localization/` — i18n (en/de via Accept-Language header)
- `public/` — Static assets served at `/` (WASM, CSS, icons)
